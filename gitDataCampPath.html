As part of my commitment to mastering version control and collaborative software development, I completed the Git Fundamentals skill track on DataCamp. This learning path includes a series of hands-on courses that build practical Git skills — from basic commands to more advanced workflows used in real-world projects.

The track consisted of the following three courses:

  1. Introduction to Git
    • Learned the basics of version control and its benefits in software projects
    • Navigated the interface (CLI) to execute Git commands
    • Managed file states including staging, saving, and committing changes
    • Compared different file versions and understood commit differences
    • Practiced reverting changes, unstaging files, and commits
  2. Intermediate Git
    • Created, renamed, modified, and deleted branches using Git CLI
    • Used branch comparison to understand differences and development progress
    • Performed merges between branches, including resolving merge situations
    • Learned to manage merge conflicts and minimize risk during collaboration
    • Cloned and synchronized projects using remote repositories
    • Used git pull, git fetch, and git push to sync local and remote branches
  3. Advanced Git
    • Mastered advanced Git merge strategies including fast-forward, recursive, squash, and octopus merges
    • Used interactive rebase and compared it to standard merge workflows
    • Practiced complex merge scenarios and reorganized commit history with Git rebase
    • Applied cherry-picking to move specific commits between branches
    • Resolved issues during cherry-pick with abort handling
    • Used Git bisect to locate bugs through binary search of commit history
    • Learned essential Git security practices by using Git Filter Repo to detect remove sensitive information from version history
    • Recovered from mistakes using Git reflog, including restoring deleted branches and lost commits
    • Worked with Git worktrees to manage multiple working directories in parallel
    • Used and managed Git submodules for modularizing repositories
    • Implemented Git LFS (Large File Storage) for handling large binary assets
    • Learned and applied principles of Trunk Based Development for modern workflows
    • Managed ETL-oriented repositories with efficient repository structuring techniques
